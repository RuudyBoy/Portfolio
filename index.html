In the report the student says they first wanted to put the image in via CSS. The main consideration for putting images as HTML or via CSS is whether or not it’s content. If it is content, then it should be in the HTML. If it’s for styling, then it should be added via CSS. The reason stated for moving to HTML was to avoid repeating styles. That’s a good thought, but you can do that simply by using classes. For example: header__img { background-size: cover; background-repeat: no-repeat; background-position: top; } header__home { background-image: url(“images/home.jpg”); } header__about{ background-image: url(“images/about.jpg”); } It’s important to note that you can add multiple classes to the same html element.
